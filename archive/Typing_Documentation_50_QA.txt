1. Что такое type hints в Python?
Ответ: Type hints — это способ аннотирования типов аргументов и возвращаемого значения функции. Например: def add(x: int, y: int) -> int. Они не влияют на выполнение, но улучшают читаемость и поддержку кода.

2. Зачем использовать mypy?
Ответ: mypy — это инструмент статической типизации Python-кода. Он анализирует аннотации и выявляет потенциальные ошибки до выполнения программы, повышая надёжность.

3. Что делает Optional в типизации?
Ответ: Optional[X] означает, что переменная может быть либо типа X, либо None. Это эквивалент Union[X, None] и полезно при необязательных аргументах.

4. Что такое Union в typing?
Ответ: Union позволяет задать несколько допустимых типов для переменной или параметра. Пример: Union[int, str] допускает как числа, так и строки.

5. Зачем использовать TypedDict?
Ответ: TypedDict позволяет аннотировать словари с фиксированной структурой и типами значений, например: {'id': int, 'name': str}. Это делает словари безопаснее и понятнее.

6. Чем отличается List[int] от list[int]?
Ответ: List[int] — старая форма записи из typing, совместимая с Python 3.6+. Начиная с Python 3.9 можно использовать встроенные generic-типы: list[int].

7. Что такое Literal?
Ответ: Literal позволяет задать ограниченное множество допустимых значений. Например: Literal['GET', 'POST'] позволяет только два конкретных строковых значения.

8. Что такое Protocol в typing?
Ответ: Protocol — это интерфейс, который описывает структуру объекта. Это позволяет использовать duck typing с типовой проверкой и без наследования от конкретного класса.

9. Что такое stub-файлы (.pyi)?
Ответ: Stub-файлы содержат только сигнатуры функций и классов без реализации. Используются для типизации сторонних библиотек без аннотаций.

10. Как работает autodoc в Sphinx?
Ответ: autodoc — расширение Sphinx, которое извлекает docstrings из Python-кода и вставляет их в документацию. Это позволяет поддерживать документацию в актуальном состоянии.
