1. Что такое REST API и его ключевые принципы?
Ответ: REST API — это архитектурный стиль для взаимодействия между клиентом и сервером по протоколу HTTP. Ключевые принципы: использование HTTP-методов (GET, POST, PUT, DELETE), ресурсы представлены через URL, отсутствует сохранение состояния (stateless), и поддерживается кеширование.

2. Чем отличается PUT от PATCH?
Ответ: PUT используется для полного обновления ресурса, в то время как PATCH применяется для частичного обновления. PUT требует полного представления ресурса, а PATCH — только изменяемые поля.

3. Что такое OpenAPI (Swagger)?
Ответ: OpenAPI — это спецификация, описывающая REST API. Она позволяет генерировать документацию, тестировать API и автоматически создавать клиентов и серверы. Swagger — один из инструментов, реализующих эту спецификацию.

4. Что такое idempotent API метод?
Ответ: Идемпотентный метод при повторном вызове не изменяет состояние сервера. Например, DELETE и PUT — идемпотентные, потому что многократное удаление или обновление одного ресурса приводит к одинаковому результату.

5. Что такое 401 и 403 статус-коды?
Ответ: 401 Unauthorized — клиент не авторизован (нужен токен или логин). 403 Forbidden — авторизация есть, но нет прав на выполнение действия.

6. Что такое Webhook?
Ответ: Webhook — это механизм, при котором сторонний сервис отправляет POST-запрос на ваш URL при наступлении события. Например, уведомление о платеже от платёжного шлюза.

7. Чем отличается REST от RPC?
Ответ: REST ориентирован на ресурсы и использует стандартные HTTP-методы. RPC (Remote Procedure Call) — это вызов удалённых процедур, часто использует POST с действием в теле запроса и не всегда следует REST-принципам.

8. Что такое API gateway?
Ответ: API gateway — это сервер-посредник, который управляет входящими запросами к микросервисам. Он выполняет маршрутизацию, аутентификацию, лимитирование, логирование и агрегацию ответов.

9. Что такое CORS и зачем он нужен?
Ответ: CORS (Cross-Origin Resource Sharing) — это механизм безопасности браузеров, ограничивающий запросы между разными доменами. Сервер должен явно разрешать такие запросы через заголовки ответа.

10. Что такое rate limiting и как его реализовать?
Ответ: Rate limiting — это ограничение количества запросов от клиента за определённый промежуток времени. Реализуется с помощью middleware, API gateway или сторонних решений, как Redis-лимитер.
