Типы данных в Python.
1. Числовые типы (Numeric Types)
Считаются неупорядоченными, так как не являются коллекциями и не поддерживают индексы.
- int (целые числа): (неизменяемый, неупорядоченный, хешируемый)
- float (вещественные числа): (неизменяемый, неупорядоченный, хешируемый)
- complex (комплексные числа): (неизменяемый, неупорядоченный, хешируемый)
- bool (логический тип): (неизменяемый, неупорядоченный, хешируемый)

2. Последовательности (Sequence Types)
- str (строки): (неизменяемый, упорядоченный, хешируемый)
- list (списки): (изменяемый, упорядоченный, нехешируемый)
- tuple (кортежи): (неизменяемый, упорядоченный, хешируемый*)
- range (диапазоны): (неизменяемый, упорядоченный, хешируемый)

3. Бинарные последовательности (Binary Sequence Types)
- bytes (байты): (неизменяемый, упорядоченный, хешируемый)
- bytearray (массив байтов): (изменяемый, упорядоченный, нехешируемый)
- memoryview (проекция памяти): (зависит**, упорядоченный, зависит**)

4. Множества (Set Types)
- set (множества): (изменяемый, неупорядоченный, нехешируемый)
- frozenset (неизменяемые множества): (неизменяемый, неупорядоченный, хешируемый)

5. Отображения (Mapping Types)
- dict (словари): (изменяемый, упорядоченный***, нехешируемый)

6. Специальные типы
- NoneType (тип объекта None): (неизменяемый, неупорядоченный, хешируемый)

Важные технические уточнения (для собеседований):
* Кортежи (tuple) хешируемы только в том случае, если все элементы внутри них 
хешируемы. Если в кортеже лежит список (например, (1, 2, [3, 4])), весь кортеж 
становится нехешируемым.
** memoryview перенимает свойства объекта, на который ссылается. Если он 
ссылается на bytes, он неизменяемый и хешируемый. Если на bytearray — изменяемый 
и нехешируемый.
*** Словари (dict) официально стали упорядоченными (сохраняют порядок добавления 
элементов) начиная с версии Python 3.7. До этого они считались неупорядоченными.

3. Что такое переменная и объект в Python?

```plaintetx
Переменная в Python — это не ячейка памяти (как в C или Java), а просто
символическое имя (строковый идентификатор), являющееся ссылкой на объект.
Переменные хранятся в специальных словарях пространств имен (например,
locals() или globals()). Важнейшее правило: переменная в Python не имеет
типа данных, тип есть только у самого объекта, на который она ссылается.

Объект (PyObject) — это базовая структура данных на уровне языка C, из которой
состоит абсолютно всё в Python (числа, строки, списки, функции и типы данных).

Под капотом каждый базовый PyObject обязательно имеет служебные атрибуты
(заголовок объекта) и само значение:

1. ob_refcnt (Счетчик ссылок / Reference count):
Это целое число, которое хранит количество переменных (и других объектов),
указывающих на этот объект в данный момент.
   - Когда мы пишем a = 10 и b = a, ob_refcnt объекта 10 становится равен 2.
   - Когда переменная удаляется (del a), переназначается или выходит из области
   видимости, счетчик уменьшается.
   - Как только ob_refcnt == 0, механизм подсчета ссылок мгновенно освобождает
   память.

2. ob_type (Указатель на тип объекта):
Это ссылка на другой объект в памяти, который определяет тип данных (например,
класс int или str). Именно благодаря этому полю Python знает, какие методы
можно применять к объекту. Это физическая основа динамической и строгой
типизации.

3. ob_size (Только для коллекций и строк - PyVarObject):
Дополнительный атрибут для объектов переменной длины (списки, строки, словари),
который хранит количество элементов внутри этого объекта.

4. Само значение (Value):
Полезная нагрузка. Для целого числа это будет C-структура с массивом цифр,
для строки — массив байтов и так далее.

Сборщик мусора (Garbage Collector):
Одного подсчета ссылок недостаточно. Если создать список A и добавить в него
список B, а в список B добавить список A, возникнет циклическая ссылка
(reference cycle). Их счетчики ссылок никогда не опустятся до нуля, даже если
удалить переменные A и B.

Поэтому сборка мусора в Python делится на два механизма:
1. Подсчет ссылок (Reference Counting): Работает всегда, в реальном времени
удаляет объекты, счетчик которых достиг нуля.
2. Generational GC (Поколенческий сборщик мусора): Фоновый процесс, который
периодически сканирует память (разделенную на 3 поколения) специально для того,
чтобы находить и уничтожать замкнутые кольцевые ссылки, которые подсчет ссылок
удалить не может.
```

4. Числа в python

```plaintext
int (Целые числа)В Python 3 тип int реализует арифметику произвольной точности (bignum arithmetic). Он математически 
точно представляет любое целое число $n \in \mathbb{Z}$, ограниченное лишь объемом доступной оперативной памяти. 
Внутренняя реализация (CPython) хранит число не как фиксированный 32- или 64-битный блок, а как массив "цифр" 
(digits) по основанию $2^{30}$ (на 64-битных системах). Таким образом, сложность базовых арифметических операций, 
таких как сложение, составляет $O(N)$, а умножения — от $O(N^2)$ до $O(N^{\log_2 3})$, где $N$ — количество $30$-битных 
блоков числа.
```
